## Overview

This repository contains a Proof of Concept (PoC) for a side-channel attack targeting WhatsApp's reaction mechanism. By measuring the precise timing between a message's server acknowledgment (Single Tick) and its delivery to the target device (Double Tick), it is possible to infer information about the target's device state.

### The Theory

The core of this side-channel lies in the **Delivery Time Delta** ($\Delta T$):

$$\Delta T = T_{delivered} - T_{sent\_to\_server}$$

1.  **Single Tick ($T_{sent\_to\_server}$):** The moment the WhatsApp server acknowledges receipt of your reaction.
2.  **Double Tick ($T_{delivered}$):** The moment the target device receives the reaction and sends back a delivery receipt.

#### Why is this a vulnerability?
Mobile operating systems (Android/iOS) handle background processes differently based on the device state:
- **Active/Foreground:** The app is open; $\Delta T$ is minimal (mostly network latency).
- **Background (Recent):** The app is in the background but not yet suspended; $\Delta T$ is slightly higher.
- **Suspended/Doze Mode:** The OS has restricted background activity to save battery. The delivery of the reaction might be delayed until a "maintenance window" or a high-priority push notification wakes up the app.

By sending a series of reactions and analyzing the distribution of $\Delta T$, an observer can potentially distinguish between an active user, an idle device, or even identify the OS type and battery optimization patterns.

## Features

- **Multi-Target Analysis (Go):** Concurrent tracking of multiple targets with round-robin reaction sending.
- **Precision Timing:** High-resolution measurement of delivery deltas.
- **Automated Plotting (Python):** Visualization tools to analyze distribution and time-series patterns.
- **Dual Implementation:** Includes both a Go engine (using `whatsmeow`) and a Python version (using `neonize`).

## Project Structure

- `main.go`: The primary Go engine for data collection. Supports multi-target and high-concurrency.
- `main.py`: Python implementation for quick testing and integrated plotting.
- `plot_results.py`: Advanced plotting script for CSV data generated by the Go engine.
- `requirements.txt`: Python dependencies.
- `go.mod`: Go module definition.

## Getting Started

### Prerequisites

- **Go** (1.25+ recommended)
- **Python 3.10+**
- A WhatsApp account for the "sender" device (requires QR pairing).

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/yourusername/wa-reaction-sidechannel-poc.git
   cd wa-reaction-sidechannel-poc
   ```

2. Install Go dependencies:
   ```bash
   go mod tidy
   ```

3. Install Python dependencies:
   ```bash
   pip install -r requirements.txt
   ```

### Usage

#### 1. Data Collection (Go)
Run the Go engine to start the measurement process:
```bash
go run main.go
```
Follow the interactive prompts to:
- Pair your device via QR code.
- Enter target phone numbers (with country code).
- Configure the number of reactions and delay.

The results will be saved to `.csv` files.

#### 2. Analysis & Visualization
Use the Python script to generate graphs from the collected data:
```bash
python plot_results.py results_*.csv
```
This will generate a `results_multi_target_plot.png` showing:
- **Distribution Histogram:** Helps identify "clusters" of response times.
- **Time Series Plot:** Shows how responsiveness changes over time (e.g., as a device enters sleep mode).
